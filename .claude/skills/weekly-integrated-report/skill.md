## Пайплайн: «Нагрузка и эффективность кураторов»

Запускай субагентов **строго последовательно** — каждый следующий получает вывод предыдущего.

---

### Шаг 1 — Анализ SLA (kln-analyst)

```
Используй агента kln-analyst. Обработай входные файлы KPI:
- docs/KPI SLA ДЗ.png — скрин таблицы ДЗ: прочитай как изображение, извлеки данные, сохрани как docs/KPI SLA ДЗ.md, удали PNG
- docs/KPI SLA ЧАТЫ.md — таблица просрочек чатов, уже в markdown: прочитай напрямую

Верни структурированный анализ нарушений SLA (порог: % просрочки > 5%).
```

Сохрани вывод — это `[KPI_АНАЛИЗ]`.

---

### Шаг 2 — Анализ нагрузки (busy-analyst)

```
Используй агента busy-analyst. Определи активные потоки на сегодняшнюю дату по файлам data/запуски-потоков.md и data/🎓 ЗАПУСКИ ПОТОКОВ - АСИНХРОННЫЕ.md. Верни таблицу нагрузки с индексом и классификацией по каждому куратору.
```

Сохрани вывод — это `[НАГРУЗКА]`.

---

### Шаг 3 — Корреляция и рекомендации (correlation-engine)

```
Используй агента correlation-engine. Вот два блока данных:

## Данные KPI:
[KPI_АНАЛИЗ]

## Данные нагрузки:
[НАГРУЗКА]

Сопоставь данные, определи паттерны (A/B/C/D), сформируй рекомендации по перераспределению потоков.
```

Сохрани вывод — это `[КОРРЕЛЯЦИЯ]`.

---

### Шаг 4 — Сохранение интегрированного отчёта

Сформируй финальный файл `reports/ГГГГ-ММ-ДД-integrated-curators-report.md` в структуре:

```markdown
# Нагрузка и эффективность кураторов — [Неделя/Месяц]

> Дата отчёта: [сегодня]  
> Подготовлено: автоматически (kln-analyst → busy-analyst → correlation-engine)

---

## Часть 1. KPI — Слабые показатели

[вставь KPI_АНАЛИЗ]

---

## Часть 2. Нагрузка — Активные потоки

[вставь НАГРУЗКА]

---

## Часть 3. Корреляция и рекомендации

[вставь КОРРЕЛЯЦИЯ]

---

## Итоговые действия

[выпиши 3–5 конкретных действий из раздела рекомендаций, сформулированных как задачи: «Передать поток X от А к Б», «Провести 1:1 с куратором В по теме CSI»]
```

---

### Шаг 5 — Подтверждение

Выведи:
- Путь к созданному файлу
- Одну строку: «X из Y кураторов требуют внимания; рекомендовано N действий»

---

**Входные данные:** `docs/KPI SLA ДЗ.png` (конвертируется в MD), `docs/KPI SLA ЧАТЫ.md`, `data/запуски-потоков.md`, `data/🎓 ЗАПУСКИ ПОТОКОВ - АСИНХРОННЫЕ.md`  
**Выходной артефакт:** `reports/ГГГГ-ММ-ДД-integrated-curators-report.md`
