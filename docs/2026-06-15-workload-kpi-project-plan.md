# Личный проект: «Нагрузка и эффективность кураторов»
**Финальный проект курса ZeroCoder — CLm03**  
Дата: 2026-06-15

---

## Описание проекта

**Я сделала проект для себя как лида команды кураторов, который помогает решить проблему ручного еженедельного анализа данных.** Раньше я вручную сопоставляла данные по нагрузке (количество потоков и студентов на куратора) с данными КЛН (KPI: SLA и CSI по ДЗ и чатам). Теперь мультиагентная система делает это автоматически: я запускаю одну команду и получаю готовый аналитический отчёт.

**Пользователь делает:** запускает скилл `/weak-report` раз в неделю.  
**На выходе получает:** структурированный отчёт с тремя слоями анализа:  
1. Кто из кураторов показывает слабые KPI-метрики  
2. Какая у них нагрузка по потокам в этот период  
3. Вывод: влияет ли перегрузка на качество, и кому из команды можно перераспределить потоки  

**Это полезно потому, что** еженедельный анализ занимал 40–60 минут ручной работы; агент делает его за 2 минуты, и я не пропускаю момент, когда куратор начинает «тонуть» под нагрузкой.

**Дальше проект можно развивать через:** автоматический запуск по расписанию (cron) каждое воскресенье вечером с пушем отчёта в Telegram.

## Архитектура системы

```
data/КЛН Кураторов — [Месяц].md         ← KPI-метрики кураторов
data/запуски-потоков.md                  ← Нагрузка: потоки, студенты
         │                    │
         ▼                    ▼
  [kln-analyst]       [busy-analyst]      ← Агенты-аналитики
         │                    │
         └──────────┬─────────┘
                    ▼
         [correlation-engine]             ← Агент корреляции
                    │
                    ▼
         [report-formatter]              ← Форматирование
                    │
                    ▼
         reports/ГГГГ-ММ-ДД-weekly-report.md
```

### Компоненты

| Компонент | Статус | Функция |
|---|---|---|
| `kln-analyst` | ✅ Готов | Читает КЛН, находит метрики ниже порога |
| `report-formatter` | ✅ Готов | Форматирует анализ в отчёт |
| `weak-report` (скилл) | ✅ Готов | Запускает kln-analyst → report-formatter |
| `busy-analyst` | ✅ Готов | Читает запуски-потоков.md + АСИНХРОННЫЕ, считает активную нагрузку, выдаёт индекс |
| `correlation-engine` | ✅ Готов | Сопоставляет нагрузку и KPI, определяет паттерны A/B/C/D, формирует рекомендации |
| `weekly-integrated-report` (скилл) | ✅ Готов | Единый пайплайн: все агенты → интегрированный отчёт |

## Критерии успешного результата

1. **Нагрузка** — агент корректно считает количество активных потоков и студентов на каждого куратора в текущем периоде
2. **Корреляция** — в отчёте есть раздел «Нагрузка vs. KPI» с выводом по каждому куратору с отклонениями
3. **Рекомендации** — конкретные предложения: у кого забрать поток, кому можно добавить (с указанием имён и потоков)
4. **Скорость** — от запуска до готового файла не более 3 минут

## Вектор монетизации навыка

Этот проект — демонстрация **автоматизации операционных задач внутри своей деятельности**.  
Навык, который здесь применяется: построение мультиагентного аналитического пайплайна на Claude Code — от сырых данных к готовому решению.

Применимость за пределами этого проекта:

- Готовый пайплайн — портфельный кейс: **«для кого»** + **«что делает»** + **«измеримый результат»**
