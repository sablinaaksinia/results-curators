# Curator Analytics Pipeline

Автоматический еженедельный анализ нагрузки и эффективности команды кураторов университета Зерокодер.

## Что делает проект

Каждую неделю система принимает два файла из базы данных:
- скриншот статистики проверки домашних заданий (PNG)
- таблицу просрочек в учебных чатах (MD)

На выходе — полноценный HTML-отчёт с рейтингами кураторов, анализом корреляции нагрузки и KPI, и конкретными рекомендациями по перераспределению потоков.

## Как это работает

Пайплайн из трёх агентов запускается одной командой `/weekly-integrated-report`:

```
docs/KPI SLA ДЗ.png  ─┐
                       ├──► kln-analyst ──► busy-analyst ──► correlation-engine ──► report.html
docs/KPI SLA ЧАТЫ.md ─┘
```

| Агент | Задача |
|---|---|
| `kln-analyst` | Конвертирует PNG в MD, анализирует SLA по ДЗ и чатам |
| `busy-analyst` | Считает активные потоки, рассчитывает индекс нагрузки |
| `correlation-engine` | Сопоставляет нагрузку с KPI, классифицирует паттерны, формирует рекомендации |

## Структура папок

```
results-curators/
├── .claude/
│   ├── agents/
│   │   ├── kln-analyst.md          # анализ KPI и SLA
│   │   ├── busy-analyst.md         # анализ нагрузки
│   │   ├── correlation-engine.md   # корреляция и рекомендации
│   │   └── report-formatter.md     # форматирование отчёта
│   └── skills/
│       └── weekly-integrated-report/
│           └── skill.md            # скилл запуска пайплайна
├── data/                           # файлы запусков потоков
├── docs/                           # входные KPI-файлы
└── reports/                        # выходные отчёты
```

## Формулы

**Индекс нагрузки:**
```
Индекс = синхронные потоки × 2 + асинхронные курсы × 1
```

**Пороги SLA:**
- `> 15%` просрочки → 🔴 Критично
- `5–15%` просрочки → ⚠️ Нарушение
- `< 5%` просрочки → ✅ Норма

**Паттерны кураторов:**
- `A` — высокая нагрузка + нарушение KPI → перераспределить потоки
- `B` — высокая нагрузка + норма KPI → риск выгорания, мониторинг
- `C` — нарушение KPI при низкой нагрузке → поведенческая проблема, 1:1
- `D` — нагрузка 0 → резерв, готов к подключению

## Запуск

```bash
# в Claude Code
/weekly-integrated-report
```

Отчёт сохраняется в `reports/ГГГГ-ММ-ДД-curators-report.html` и открывается в браузере:

```bash
python3 -m http.server 8080
# → http://localhost:8080/reports/
```
