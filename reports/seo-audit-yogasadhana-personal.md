### 1.3 Семантическая структура HTML

**🔴 КРИТИЧЕСКАЯ ПРОБЛЕМА: Нет ни одного тега H1–H6 на странице.**

Tilda использует `<div class="t-title">` вместо семантических заголовков. Для Google это означает, что страница не имеет иерархии заголовков. Поисковые роботы не могут определить главную тему страницы.

Реальная структура «заголовков» через классы Tilda (отображается как заголовки визуально, но не является ими семантически):

| Визуальный заголовок | CSS-класс Tilda | Семантический тег | Проблема |
|---|---|---|---|
| "ИНДИВИДУАЛЬНЫЙ РЕТРИТ" | `t-title t-title_md` | `<DIV>` | H1 отсутствует |
| "Что будет на ретрите" | `t-title t-title_xxl` | `<DIV>` | H2 должен быть |
| "Все это в окружении прекрасной природы" | `t-title t-title_lg` | `<DIV>` | H2/H3 должен быть |
| "Кто проводит ретрит" | `t-section__title t-title_xs` | `<DIV>` | H2 должен быть |
| "Где будет проходить ретрит" | `t-title t-title_lg` | `<DIV>` | H2 должен быть |
| "Как это было" | `t-title t-title_lg` | `<DIV>` | H2 должен быть |

**Дополнительная проблема:** Tilda разбивает заголовки на отдельные `<span>` для анимации. Например, фраза "Что будет на ретрите" разбита на `<span>"Что будет на"</span>` и `<span>"ретрите"</span>` — поисковый робот видит бессмысленные фрагменты.

**Рекомендации для Tilda:**
1. Перейти на блоки с поддержкой SEO-заголовков (настройки блока → HTML тег заголовка)
2. В настройках каждого текстового блока в Tilda установить тег H1/H2/H3 вместо DIV
3. Главный заголовок страницы обязательно пометить как H1
