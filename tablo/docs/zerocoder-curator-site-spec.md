# Техническая документация: Сайт «Куратор Zerocoder»

**Тип проекта:** Одностраничный лендинг (SPA-страница)  
**Стек:** HTML5 + CSS3 + Vanilla JS  
**Формат поставки:** Один файл `index.html` — открывается в браузере без сервера  
**Назначение:** Онбординг новых кураторов, справочник правил и инструментов

---

## 1. Архитектура файла

```
index.html
├── <head>           — метатеги, шрифты, встроенные стили (<style>)
└── <body>
    ├── <nav>        — фиксированная навигация с якорями
    ├── #hero        — Hero-секция
    ├── #tasks       — Задачи куратора
    ├── #requirements — Требования и правила
    ├── #tools       — Инструменты работы
    ├── #onboarding  — План онбординга
    └── <footer>     — Контакты / копирайт
```

Весь CSS — в теге `<style>` внутри `<head>`.  
Весь JS — в теге `<script>` перед `</body>`.  
Никаких внешних зависимостей, кроме Google Fonts (подключается через CDN).

## 2. Дизайн-система

### Цветовая палитра
| Переменная        | Значение    | Назначение                      |
|-------------------|-------------|----------------------------------|
| `--color-brand`   | `#5B4CF5`   | Акцент, кнопки, иконки           |
| `--color-dark`    | `#1A1A2E`   | Фон hero, заголовки              |
| `--color-surface` | `#F7F7FC`   | Фон секций (чередование)         |

| `--color-text`    | `#2D2D2D`   | Основной текст                   |

| `--color-success` | `#10B981`   | Иконки выполненных шагов         |

### Типографика
- Шрифт: `Inter` (Google Fonts), fallback: `system-ui, sans-serif`
- H1: 48px / weight 700
- H2: 32px / weight 600
- H3: 20px / weight 600
- Body: 16px / weight 400 / line-height 1.6

### Сетка
- Максимальная ширина контейнера: `1100px`
- Отступы секций: `padding: 80px 24px`
- Мобильный брейкпоинт: `768px` (одна колонка)

## 3. Навигация

Фиксированная шапка `position: fixed; top: 0` с прозрачным фоном, который становится белым при скролле (через JS).

```
[Логотип: "Куратор Zerocoder"]   [Задачи] [Требования] [Инструменты] [Онбординг]
```

- Клик по пункту → плавный скролл к секции (`scroll-behavior: smooth`)
- Активный пункт подсвечивается `--color-brand` при попадании секции в viewport (IntersectionObserver)

### 4.3 Требования `#requirements`

**Фон:** `--color-surface`  
**Заголовок H2:** «Правила и стандарты работы»

Два блока рядом (flex, gap 40px), на мобиле — столбец.

#### Блок 1: «Качество»
Список с галочками `✅`:
- Проверять ДЗ строго по критериям, без субъективных оценок
- Давать конструктивную обратную связь — не «неправильно», а «как сделать правильно»
- Не снижать балл без объяснения причины
- Использовать единые формулировки, согласованные в команде
- При сомнениях — уточнять у лид-куратора, не додумывать самостоятельно

#### Блок 2: «Сроки и дисциплина»
Список с галочками `✅`:
- Проверить ДЗ в течение **24 часов** с момента сдачи
- Ответить в чате в течение **2 часов** в рабочее время
- Рабочие часы: уточняются при онбординге (индивидуально)
- Сообщать лид-куратору об отсутствии минимум за **24 часа**
- Не пропускать плановые встречи команды без уведомления

#### Нижний блок — «Важно помнить» (отдельная выделенная карточка):
```
⚠️  Куратор — лицо университета для студента.
Тон общения — доброжелательный и профессиональный в любой ситуации.
Конфликтные ситуации — эскалировать лид-куратору, не решать в одиночку.
```
Оформление: карточка с левой цветной полосой `--color-brand`, фон светло-сиреневый.

### 4.5 План онбординга `#onboarding`

**Фон:** `--color-surface`  
**Заголовок H2:** «Как стать куратором за 2 недели»

Вертикальный таймлайн (timeline):
- Вертикальная линия по центру слева
- Каждый шаг — блок с номером, заголовком, описанием и сроком

#### Шаги:

```
День 1–2: Знакомство с командой и процессами
  ├─ Встреча с лид-куратором
  ├─ Доступ к инструментам (Telegram, LMS, Drive)
  └─ Изучение базы знаний: критерии, регламенты, FAQ

День 3–4: Погружение в учебный контент
  ├─ Прохождение ключевых модулей курса (как студент)
  ├─ Знакомство со структурой ДЗ
  └─ Изучение примеров проверенных ДЗ (хорошие и плохие)

День 5–7: Первые проверки с наставником
  ├─ Куратор проверяет ДЗ самостоятельно
  ├─ Лид-куратор даёт обратную связь по каждой проверке
  └─ Совместный разбор спорных кейсов

День 8–10: Самостоятельная работа под контролем
  ├─ Куратор работает в штатном режиме
  ├─ Ежедневный чек-ин с лид-куратором (15 мин)
  └─ Первые ответы в учебном чате

День 11–14: Полная интеграция
  ├─ Закрытие онбординга: встреча с лид-куратором
  ├─ Финальная обратная связь по первым двум неделям
  └─ Куратор переходит в самостоятельный режим работы ✅
```

Каждый шаг оформлен как карточка с:
- Нумерованным кружком (фон `--color-brand`, белый номер)
- Заголовком этапа (H3)
- Списком действий
- Бейджем со сроком (pill-badge)
